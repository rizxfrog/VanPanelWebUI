# A2赛题：面向麒麟操作系统的安全智能运维 Agent - 前端架构与开发规划文档

## 一、 赛题背景与核心目标分析

本赛题（第十五届中国软件杯 A2组）的核心难点不在于“能否让大模型执行命令”，而在于**“解决 AI 推理的不可控性”**。作为系统的直接展示面，前端（基于现有的 `VanPanelWebUI` / Vben Admin 架构）必须承担起将底层安全机制“可视化、可交互、可溯源”的重任。

前端的核心目标：
1. **透明性（See）**：将 Agent 的“感知-推理-执行”思维链全息展示。
2. **安全性（Safe）**：将底层安全拦截、越权操作可视化警告并提供“Human-in-the-loop”（人工审批）闭环。
3. **沉浸式交互（Interact）**：摆脱干瘪纯文本，使用多模态卡片化视图展示系统状态。
4. **信创适配验证（Deploy）**：确保编译产物在龙芯架构+麒麟服务器 V11 及其自带浏览器上流畅运行。

---

## 二、 前端技术栈与架构规范

本项目基于现有的 `VanPanelWebUI` monorepo 架构开发，继承和扩展以下技术：

| 领域 | 技术选型 | 说明 |
| :--- | :--- | :--- |
| **底层框架** | Vue 3.4+ (Composition API), Vite, TypeScript | 保持现有 Vben 架构，提供极高开发效率与运行性能 |
| **UI 组件库** | Ant Design Vue 4.x | 构建企业级运维管理后台核心组件 |
| **状态管理** | Pinia | 管理全局用户状态、安全拦截状态、系统监控数据流 |
| **通信协议** | SSE (Server-Sent Events) + Axios | **重点**：使用 SSE 对接大模型对话流，打字机体验；Axios 处理常规 CRUD |
| **可视化** | ECharts (vue-echarts) | 用于 OS 感知大屏（CPU、内存、网络IO实时折线/仪表盘） |
| **Markdown渲染** | marked / md-editor-v3 | 用于渲染大模型返回的带有代码块、表格的自然语言文本 |

## 三、 核心需要新增与改造的模块 (What & How)

我们将在 `apps/web-antd/src/views/` 目录下进行大规模的业务模块扩充，主要包括以下 5 大核心模块：

### 1. 智能运维控制台 (Agent Chat Console)
**目标**：替代传统终端的智能对话主窗口，支持流式交互与多模态展现。
*   **规划路径**：新建 `views/agent-console`
*   **改造要点**：
    *   **流式对话流组件**：封装 `ChatBubble` 组件，对接后端 SSE 流，支持 Markdown 解析与代码高亮。
    *   **思维链 (CoT) 折叠面板**：每条回复下方嵌入 `<Collapse>`，展示 MCP 工具调用日志（如：`Action: os_query`, `Result: {"cpu_usage": 90%}`）。
    *   **动态卡片渲染机制**：解析后端返回的特定 JSON 指令，将纯文本转换为 Ant Design Vue 的 `<Table>` 或 `<Card>`（例如：进程列表不仅是文本，而是一个包含“终止进程”按钮的交互表格）。

### 2. 系统实时感知看板 (OS Dashboard)
**目标**：将大模型感知到的操作系统底层数据进行高维可视化。
*   **规划路径**：改造 `views/dashboard/analytics` 或新建 `views/os-dashboard`
*   **改造要点**：
    *   集成 Echarts，设计深色模式下极具科技感的仪表盘，监控项包括：CPU负载、内存碎片、磁盘IO、TCP连接数。
    *   **联动交互**：实现与对话终端的全局状态联动（如通过 Agent 结束某个高占用进程后，大屏 CPU 折线图实时动态回落）。

### 3. 安全护栏与人工审批中心 (Security & Human-in-the-loop)
**目标**：赛题最核心的“解决不可控性”的体现，防止误删库。
*   **规划路径**：新建 `views/security-approval` 以及全局拦截弹窗。
*   **改造要点**：
    *   **拦截警报气泡**：当检测到 rm、chmod 777 或 Prompt 注入时，对话框弹出醒目红色 `Alert` 提示：“⛔ 操作被安全护栏拦截”。
    *   **高危指令审批工作流**：在对话框内渲染“审批卡片”，当 Agent 评估为高危但合理的操作时，卡片展示 `Accept` 和 `Reject` 按钮，并要求管理员二次输入密码或调用指纹/UKey，方可放行。

### 4. 运维全链路溯源审计 (Audit Trail)
**目标**：满足赛题“支持异常回溯”与“推理链路溯源”的闭环日志需求。
*   **规划路径**：新建 `views/audit-log`
*   **改造要点**：
    *   基于 `BasicTable` 封装复杂的查询列表页面。
    *   记录字段包含：触发时间、操作者 (Agent/Admin)、原始指令、调用的 MCP 插件、安全拦截状态、执行结果等。
    *   提供“报表导出”功能，以 PDF 或 Excel 形式导出，充实初赛需提交的《测试报告》数据。

### 5. MCP 插件管理台 (MCP Plugin Manager)
**目标**：体现赛题要求的“运维插件化架构”。
*   **规划路径**：新建 `views/mcp-manager`
*   **改造要点**：
    *   开发应用商店风格的卡片列表（`Card.Grid`），展示当前系统加载的 MCP Tools（如：“日志分析助手”、“僵尸进程清理器”、“网络限流工具”）。
    *   支持可视化开启/禁用插件，支持设置插件的提权限制（模拟最小权限代理执行原则）。

## 五、 后续实施里程碑建议

为了高效完成初赛作品并获得评委青睐，建议分阶段实施：

*   **Phase 1: 原型与静态结构跑通 (第1-2周)**
    *   完成上述所有路由及空页面的创建。
    *   完成 `Agent Chat Console` 的静态 UI 搭建，跑通模拟的流式打字机效果。
*   **Phase 2: 核心链路联调 (第2-4周)**
    *   前端对接真实后端的 SSE 接口。
    *   实现 **“接收指令 -> MCP工具展示 -> 思维链展示 -> 结果展示”** 的主干功能。
*   **Phase 3: 爆点与安全护栏打磨 (第5-6周)**
    *   实装高危命令（如 `rm -rf`）前端拦截的红牌警告动效。
    *   实装 Human-in-the-loop 人工审批拦截卡片。
*   **Phase 4: 信创环境测试与产物打包 (第7周)**
    *   打出前端 release 包，部署至 **龙芯 + 麒麟V11** 服务器，使用自带浏览器进行压力与兼容性测试。
    *   录制演示视频，准备初赛文档。
